## Часть 1 — Асимптотический анализ

**Ответы в `complexity.md`:**

| Фрагмент | Сложность по времени | Сложность по памяти |
|----------|----------------------|---------------------|
| A        | O(n)                 | O(1)                |
| B        | O(n²)                | O(1)                |
| C        | O(n)                 | O(1)                |
| D        | O(2ⁿ)                | O(n)                |

> **Пояснение к фрагменту C:** два отдельных цикла по n дают `n + n = 2n`, но константа отбрасывается → O(n).
>
> **Пояснение к фрагменту D:** дерево рекурсии имеет глубину n и каждый уровень удваивает число вызовов → O(2ⁿ) по времени. Память — O(n), потому что в любой момент в стеке находится не более n вызовов (глубина дерева, а не общее число узлов).

**Ответ на вопрос про `n² + n`:**

```
В асимптотическом анализе нас интересует поведение при больших n.
При n = 1000: n² = 1 000 000, n = 1 000.
Слагаемое n составляет 0.1% от n² — оно не влияет на скорость роста.
Мы оставляем доминирующий член, который определяет класс сложности.
```

## Часть 2 — Слияние отсортированных массивов

```go
// merge_sorted.go

package main

import "fmt"

// Идея: начинаем с конца обоих массивов, заполняем nums1 с конца.
// Так не нужен дополнительный массив — пишем в уже выделенное место.
func merge(nums1 []int, m int, nums2 []int, n int) {
    p1 := m - 1     // указатель на последний реальный элемент nums1
    p2 := n - 1     // указатель на последний элемент nums2
    p := m + n - 1  // указатель на позицию для записи (конец nums1)

    for p1 >= 0 && p2 >= 0 {
        if nums1[p1] > nums2[p2] {
            nums1[p] = nums1[p1]
            p1--
        } else {
            nums1[p] = nums2[p2]
            p2--
        }
        p--
    }

    // Если в nums2 остались элементы — дописываем их
    for p2 >= 0 {
        nums1[p] = nums2[p2]
        p2--
        p--
    }
    // Если остались элементы в nums1 — они уже на своих местах
}

func main() {
    nums1 := []int{1, 2, 3, 0, 0, 0}
    merge(nums1, 3, []int{2, 5, 6}, 3)
    fmt.Println(nums1) // [1 2 2 3 5 6]
}

// Сложность по времени: O(n + m) — каждый элемент обрабатывается ровно раз
// Сложность по памяти: O(1) — пишем прямо в nums1, доп. память не нужна
```

## Часть 3 — Палиндром

```go
// palindrome.go

package main

import (
    "fmt"
    "unicode"
)

func isPalindrome(s string) bool {
    // Преобразуем строку в срез рун для корректной работы с unicode
    runes := []rune(s)
    left := 0
    right := len(runes) - 1

    for left < right {
        // Пропускаем не-буквы и не-цифры слева
        for left < right && !unicode.IsLetter(runes[left]) && !unicode.IsDigit(runes[left]) {
            left++
        }
        // Пропускаем не-буквы и не-цифры справа
        for left < right && !unicode.IsLetter(runes[right]) && !unicode.IsDigit(runes[right]) {
            right--
        }

        // Сравниваем символы без учёта регистра
        if unicode.ToLower(runes[left]) != unicode.ToLower(runes[right]) {
            return false
        }

        left++
        right--
    }

    return true
}

func main() {
    fmt.Println(isPalindrome("A man, a plan, a canal: Panama")) // true
    fmt.Println(isPalindrome("race a car"))                     // false
    fmt.Println(isPalindrome(" "))                              // true
}

// Сложность по времени: O(n) — каждый символ посещается не более одного раза
// Сложность по памяти: O(n) — храним срез рун (можно сделать O(1) работая с байтами ASCII)
```

## Часть 4 — Two Sum

```go
// two_sum.go

package main

import "fmt"

// Идея: проходим массив один раз.
// Для каждого числа вычисляем, какое число нам нужно (target - nums[i]).
// Ищем это число в хэш-таблице. Если нашли — готово.
// Если не нашли — сохраняем текущее число и его индекс в таблицу.
//
// В хэш-таблице хранятся: ключ = значение числа, значение = его индекс.
// Это даёт O(1) на поиск, итого O(n) за весь проход.
func twoSum(nums []int, target int) []int {
    seen := make(map[int]int) // число → индекс

    for i, num := range nums {
        complement := target - num

        if j, ok := seen[complement]; ok {
            return []int{j, i}
        }

        seen[num] = i
    }

    return nil // по условию задачи всегда есть решение
}

func main() {
    fmt.Println(twoSum([]int{2, 7, 11, 15}, 9))  // [0 1]
    fmt.Println(twoSum([]int{3, 2, 4}, 6))         // [1 2]
    fmt.Println(twoSum([]int{3, 3}, 6))             // [0 1]
}

// Сложность по времени: O(n) — один проход по массиву, O(1) на каждый поиск в map
// Сложность по памяти: O(n) — в худшем случае храним все элементы в map
```

## Часть 5 — Top K Frequent Elements

```go
// top_k_frequent.go

package main

import "fmt"

// Идея (bucket sort):
// 1. Считаем частоту каждого числа через map — O(n).
// 2. Создаём массив buckets размером n+1, где индекс = частота.
//    buckets[freq] содержит список чисел с такой частотой.
// 3. Проходим buckets с конца (самые частые) и собираем k чисел.
//
// Итого O(n) — без сортировки!
func topKFrequent(nums []int, k int) []int {
    // Шаг 1: подсчёт частот
    freq := make(map[int]int)
    for _, num := range nums {
        freq[num]++
    }

    // Шаг 2: bucket sort — индекс = частота
    buckets := make([][]int, len(nums)+1)
    for num, count := range freq {
        buckets[count] = append(buckets[count], num)
    }

    // Шаг 3: собираем k самых частых, идём с конца
    result := make([]int, 0, k)
    for i := len(buckets) - 1; i >= 0 && len(result) < k; i-- {
        for _, num := range buckets[i] {
            result = append(result, num)
            if len(result) == k {
                break
            }
        }
    }

    return result
}

func main() {
    fmt.Println(topKFrequent([]int{1, 1, 1, 2, 2, 3}, 2)) // [1 2]
    fmt.Println(topKFrequent([]int{1}, 1))                  // [1]
}

// Сложность по времени: O(n) — подсчёт частот O(n) + заполнение buckets O(n) + сбор результата O(n)
// Сложность по памяти: O(n) — map частот + массив buckets
```

## Часть 6 — Итоговый файл

```markdown
# summary.md

## 1. Два указателя: навстречу vs в одну сторону

**Навстречу** (left с начала, right с конца):
Подходит когда нужно проверить симметрию или найти пару элементов
с определённой суммой в отсортированном массиве.
Примеры: палиндром, Two Sum в отсортированном массиве, задача с контейнером воды.

**В одну сторону** (оба идут слева, с разной скоростью):
Подходит для работы со связными списками (поиск середины, цикл),
а также для задач типа "скользящее окно".
Примеры: поиск цикла в списке (быстрый и медленный указатель), удаление дубликатов.

## 2. Хэш-таблица vs сортировка + два указателя для Two Sum

Сортировка + два указателя: O(n log n) по времени (из-за сортировки).
Хэш-таблица: O(n) по времени — один проход, каждый поиск O(1).

Хэш-таблица быстрее, потому что не требует упорядочивания данных.
Цена — дополнительная память O(n) для хранения таблицы.

## 3. Амортизированная сложность

Амортизированная сложность — это средняя стоимость операции,
усреднённая по большому числу операций.

Пример: динамический массив (slice в Go).
Добавление элемента обычно O(1), но иногда массив переполняется —
тогда создаётся новый массив вдвое больше и все элементы копируются: O(n).
Но такое копирование происходит редко. Если разложить стоимость
по всем операциям вставки, получим O(1) амортизированно.

## 4. Пространственная сложность рекурсии

Каждый рекурсивный вызов добавляет фрейм в стек вызовов.
Стек занимает память пропорционально глубине рекурсии.

Пример: рекурсивный обход дерева с глубиной n → O(n) памяти.
При этом само дерево может иметь 2^n узлов — но в стеке одновременно
находится только текущая ветка от корня до листа (глубина, а не ширина).

Поэтому рекурсия редко бывает O(1) по памяти — даже если кажется,
что алгоритм "не создаёт дополнительных структур".
```

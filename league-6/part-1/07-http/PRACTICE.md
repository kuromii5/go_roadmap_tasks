# Solution — HTTP

---

## Адресация

**1. Что такое IP-адрес?**

Уникальный идентификатор компьютера в интернете. Как номер квартиры — чтобы отправить данные нужному устройству, нужно знать его адрес.

**2. Что такое порт?**

На одном компьютере работает множество программ одновременно. IP-адрес приводит на нужный компьютер, а порт — к конкретной программе на этом компьютере. Например, порт 80 — веб-сервер, порт 443 — HTTPS. Без порта непонятно какая из программ должна обработать запрос.

**3. Что такое домен?**

Читаемый псевдоним для IP-адреса и порта. Запомнить `youtube.com` проще чем `142.250.74.78:443`. DNS переводит домен в IP-адрес при каждом обращении.

---

## Клиент и сервер

**4. Клиент и сервер:**

Клиент — программа, которая запрашивает услугу (браузер, мобильное приложение). Сервер — программа, которая обслуживает эти запросы и возвращает ответы.

**5. Кто начинает первым?**

Всегда клиент. Сервер только ждёт и отвечает — он никогда не начинает разговор первым.

**6. Схема общения по HTTP:**

Клиент отправляет запрос → сервер получает, обрабатывает → сервер отправляет ответ → клиент получает ответ. Один цикл — один диалог.

---

## HTTP-запрос

**7. Что такое HTTP-метод?**

Слово-глагол, которое говорит серверу что именно нужно сделать. Без метода сервер не знал бы — ты хочешь получить данные, создать новые, обновить или удалить.

**8. Основные методы:**

- `GET` — получить данные
- `POST` — создать новое
- `PUT` — обновить существующее
- `DELETE` — удалить

**9. Что такое путь (endpoint)?**

На сервере много разных функций — для заказов, истории, скидок и так далее. Путь указывает к какой именно функции обращается запрос. Например `/orders` — функция для работы с заказами, `/discounts` — для скидок.

**10. Что такое тело запроса?**

Секция запроса где передаются данные — например информация о новом заказе. Используется в `POST` и `PUT` — там когда нужно что-то передать серверу. В `GET` и `DELETE` тела обычно нет.

## Форматы данных

**11. Что такое JSON?**

Стандартный формат для передачи данных в интернете. Данные представлены в виде пар «ключ: значение», понятных любому языку программирования. Современные API используют JSON потому что его легко читать человеку и легко разбирать программе.

Пример:
```json
{
  "pizza": "Маргарита",
  "size": "большая",
  "quantity": 2
}
```

---

## HTTP-ответ

**12. Что сервер возвращает в ответ?**

HTTP-ответ — объект с результатом обработки запроса. Содержит статус (успех или ошибка) и тело с данными в формате JSON. Например в ответ на запрос видео YouTube сервер вернёт название, описание, лайки, просмотры.

**13. Как сервер знает куда отправить ответ?**

При получении запроса сервер запоминает откуда он пришёл — IP-адрес и порт клиента. Ответ отправляется обратно на тот же адрес.

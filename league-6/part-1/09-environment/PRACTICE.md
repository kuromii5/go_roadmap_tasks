# Практика — Конфигурация

Возьми консольный магазин из предыдущего задания и добавь в него конфигурацию через библиотеку `cleanenv`.

---

## Шаг 1 — Установи библиотеку

```bash
go get github.com/ilyakaznacheev/cleanenv
```

---

## Шаг 2 — Создай config.yml

Создай файл `config.yml` в корне проекта:

```yaml
app_name: "Консольный магазин"
currency: "руб."
max_products: 100
```

---

## Шаг 3 — Опиши структуру конфигурации

Создай `internal/config/config.go`. Структура должна:
- Иметь теги `yaml` для связи с полями из файла
- Поле `AppName` — обязательное (тег `env-required:"true"`)
- Поля `Currency` и `MaxProducts` — с дефолтными значениями (тег `env-default`)

---

## Шаг 4 — Загрузи конфиг по динамическому пути

Напиши функцию `MustLoad() *Config` которая:
- Определяет путь до `config.yml` **динамически** — через `os.Executable()`, а не захардкоженной строкой `"config.yml"`
- Читает файл через `cleanenv.ReadConfig`
- При любой ошибке — завершает программу с `log.Fatal`
- Возвращает `*Config`

> `os.Executable()` возвращает путь до запущенного бинарника. Используй `filepath.Dir` чтобы получить папку, затем `filepath.Join` чтобы склеить с `"config.yml"`.

---

## Шаг 5 — Поддержка переменных окружения

`cleanenv` умеет читать значения не только из YAML, но и из переменных окружения. Добавь тег `env` к каждому полю структуры.

Проверь что переменная окружения перекрывает значение из файла:

```bash
APP_NAME="Мой магазин" go run ./cmd/
# Должно вывести: Добро пожаловать в Мой магазин
```

---

## Шаг 6 — Используй конфиг в приложении

Подключи конфиг в `main.go`:
- При запуске вывести: `"Добро пожаловать в <AppName>"`
- В команде `list` — выводить цены с валютой из конфига
- В команде `add` — если товаров уже `MaxProducts`, выводить ошибку и не добавлять

# Практика — Логирование

Возьми консольный магазин с конфигурацией из предыдущего задания и добавь логирование через `logrus`.

---

## Шаг 1 — Установи библиотеку

```bash
go get github.com/sirupsen/logrus
```

---

## Шаг 2 — Создай пакет логера

Создай файл `internal/logger/logger.go`.

Напиши функцию `New(level string) *logrus.Logger` которая:
- Создаёт новый экземпляр логера через `logrus.New()`
- Устанавливает уровень логирования из переданной строки (`"debug"`, `"info"`, `"warn"`, `"error"`)
- Устанавливает текстовый формат вывода
- Возвращает готовый логер

> Для парсинга уровня из строки используй `logrus.ParseLevel`

---

## Шаг 3 — Добавь уровень логирования в конфиг

Добавь в `config.yml` поле `log_level` и в структуру `Config` соответствующее поле с дефолтным значением `"info"`.

```yaml
app_name: "Консольный магазин"
currency: "руб."
max_products: 100
log_level: "info"
```

---

## Шаг 4 — Подключи логер в main

В `main.go` создай логер через `logger.New(cfg.LogLevel)` и логируй события приложения:

- При старте — `Info` с именем приложения
- При добавлении товара — `Info` с названием и ценой
- При попытке добавить товар сверх лимита — `Warn`
- При неверном вводе (неверная цена или количество) — `Error` с текстом ошибки
- При выходе — `Info`

---

## Шаг 5 — Включи debug-режим

Поменяй в `config.yml` значение `log_level` на `"debug"`.

Добавь `Debug`-логи которые показывают промежуточные шаги — например текущее количество товаров после каждой команды `add`.

Убедись что при `log_level: "info"` эти строки не выводятся, а при `"debug"` — выводятся.

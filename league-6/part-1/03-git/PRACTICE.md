# Практика — Базовый Git

---

## Часть 1 — Первый репозиторий

1. Создай публичный репозиторий на GitHub с именем `git-practice` и README.
2. Клонируй его к себе на машину.

> **Перед клонированием** убедись что у тебя настроен SSH-ключ и добавлен в GitHub — иначе при push придётся каждый раз вводить логин и пароль. Клонируй через SSH-ссылку (начинается с `git@github.com:`), а не HTTPS.
3. Настрой имя и email в Git.
4. Создай файл `about.txt`, напиши в нём пару строк о себе.
5. Добавь файл в индекс и сделай коммит с сообщением `add about.txt`.
6. Отправь изменения на GitHub. Убедись что файл появился в репозитории.

---

## Часть 2 — Несколько коммитов и история

1. Создай файл `notes.md` — тремя **отдельными коммитами** добавь в него:
    - заголовок `# Мои заметки`
    - секцию `## Git` с парой строк о командах git
    - секцию `## Go` с парой строк о языке

2. Посмотри историю коммитов командой `git log --oneline`.

3. Ответь в `about.txt` на вопросы и запушь:
    - Сколько коммитов сейчас в репозитории?
    - Какой хеш у самого первого коммита (первые 7 символов)?

---

## Часть 3 — Ветки

1. Создай ветку `feature/contacts` и переключись на неё.
2. Создай файл `contacts.md` со ссылкой на твой GitHub профиль. Закоммить.
3. Запушь ветку на GitHub. Проверь что она появилась в интерфейсе.
4. Переключись обратно на `main` и слей ветку `feature/contacts` в `main`.
5. Запушь `main` на GitHub.

---

## Часть 4 — Конфликт

1. В ветке `main` измени первую строку `about.txt` и закоммить.
2. Создай ветку `feature/update-about`, измени ту же первую строку `about.txt` — но по-другому. Закоммить.
3. Попробуй слить `feature/update-about` в `main`. Получи конфликт.
4. Открой файл, разреши конфликт вручную (убери маркеры, оставь итоговый текст).
5. Закоммить разрешение и запушь.

---

## Часть 5 — Откат изменений

1. Добавь в `notes.md` строку «это ошибочная запись». Закоммить.
2. Отмени этот коммит через `git revert`. Запушь.
3. Создай два файла: `needed.txt` и `unwanted.txt`. Добавь оба в индекс (`git add .`).
4. Убери `unwanted.txt` из индекса не удаляя файл.
5. Закоммить только `needed.txt`. Запушь.

---

## Часть 6 — Pull Request

1. Создай ветку `feature/summary`, добавь в `notes.md` секцию `## Итог` с 2–3 строками выводов. Закоммить и запушь ветку.
2. Создай Pull Request на GitHub: заголовок, описание что сделал.
3. Посмотри вкладку **Files changed** — убедись что изменения правильные.
4. Смержи PR через интерфейс GitHub.
5. Подтяни изменения в локальный `main` (`git pull`).

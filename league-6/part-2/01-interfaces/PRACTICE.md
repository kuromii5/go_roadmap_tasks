## Задача 1 — Masker

Ты пишешь утилиту для маскировки персональных данных перед логированием.

Создай интерфейс:

```go
type Masker interface {
    Mask() string
}
```

Реализуй три типа, каждый из которых маскирует данные по своим правилам:

| Тип          | Вход                  | Выход              |
|--------------|-----------------------|--------------------|
| `Email`      | `alex@gmail.com`      | `a***@gmail.com`   |
| `Phone`      | `+79991234567`        | `+7***4567`        |
| `CardNumber` | `1234567812345678`    | `****5678`         |

Напиши функцию:

```go
func PrintMasked(m Masker)
```

Которая просто выводит результат `Mask()` в stdout.

**Ожидаемый вывод `main`:**

```
a***@gmail.com
+7***4567
****5678
```

**Требования:**

- Каждый тип — `string` (например `type Email string`)
- Маскировка через работу со строками, без регулярок
- Функция `PrintMasked` принимает интерфейс, а не конкретный тип

## Задача 2 — Дополни код

Ниже — готовый код. Он **не компилируется**, потому что не хватает типа и его методов.

Твоя задача: **дописать только недостающую часть**, чтобы программа скомпилировалась и выдала ожидаемый вывод.

```go
package main

import "fmt"

type Validator interface {
    Validate() bool
    ErrorMessage() string
}

func Check(v Validator) string {
    if v.Validate() {
        return "✅ OK"
    }
    return "❌ " + v.ErrorMessage()
}

func main() {
    ages := []AgeInput{
        {Value: 25},
        {Value: -3},
        {Value: 0},
        {Value: 150},
    }

    for _, a := range ages {
        fmt.Println(Check(a))
    }
}
```

**Ожидаемый вывод:**

```
✅ OK
❌ возраст не может быть отрицательным
❌ возраст не может быть нулевым
❌ возраст не может быть больше 130
```

**Требования:**

- Не меняй существующий код — только допиши тип `AgeInput` и его методы
- Подумай, как один `ErrorMessage()` может возвращать разные сообщения в зависимости от значения

## Задача 3 — Stringify

Дан набор значений разных типов. Некоторые из них реализуют интерфейс `fmt.Stringer`, некоторые — нет.

Напиши функцию:

```go
func Stringify(values []any) []string
```

Которая проходит по слайсу и:
- если элемент реализует `fmt.Stringer` — вызывает `.String()` и добавляет результат
- если нет — добавляет строку `"unknown"`

### Вспомогательные типы (создай их):

```go
type Color struct {
    R, G, B uint8
}
// Реализует fmt.Stringer
// String() возвращает формат "rgb(R, G, B)"

type Point struct {
    X, Y int
}
// НЕ реализует fmt.Stringer
```

**Пример использования в `main`:**

```go
func main() {
    values := []any{
        Color{255, 0, 128},
        42,
        Point{1, 2},
        Color{0, 0, 0},
        "hello",
    }

    result := Stringify(values)
    for _, s := range result {
        fmt.Println(s)
    }
}
```

**Ожидаемый вывод:**

```
rgb(255, 0, 128)
unknown
unknown
rgb(0, 0, 0)
unknown
```

**Требования:**

- Используй type assertion для проверки на `fmt.Stringer` (не type switch)
- `"hello"` — это `string`, он **не** реализует `fmt.Stringer`, поэтому тоже `"unknown"`
- Функция ничего не знает о конкретных типах — только про интерфейс

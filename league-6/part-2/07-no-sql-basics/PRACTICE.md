## Задача 2 — SQL vs Document

Предметная область: **блог**. Есть авторы, посты, комментарии, теги. У поста может быть много тегов, у тега — много постов.

**Шаг 1 — SQL-схема:**

Спроектируй схему в **SQL** — таблицы, колонки, PK/FK. Не забудь про many-to-many для тегов.

**Шаг 2 — Document-схема:**

Спроектируй ту же модель в **document store** (MongoDB-стиль). Как будет выглядеть JSON-документ поста?

**Шаг 3 — Сравни:**

Ответь на вопросы:

1. Автор сменил никнейм. Где проще обновить — SQL или document? Почему?
2. Нужно получить пост со всеми комментариями за один запрос. Где проще?
3. Нужно найти все посты с тегом `"go"`. Где эффективнее?
4. У постов появилось новое поле `reading_time`. Где проще добавить?
5. Нужна гарантия, что комментарий не потеряется при одновременной записи. Где надёжнее из коробки?

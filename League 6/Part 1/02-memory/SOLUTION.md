# Solution — Память и устройство компьютера

---

## Устройство компьютера

**Основные элементы и их роль:**

- **CPU (процессор)** — выполняет команды программ. Читает инструкцию из памяти, выполняет её, записывает результат. Всё что происходит в программе — делает CPU
- **RAM (оперативная память)** — временное хранилище для запущенных программ и их данных. Быстрая, но теряет всё при отключении питания
- **HDD / SSD (жёсткий диск)** — постоянное хранилище. Медленнее RAM, но данные не исчезают при выключении
- **Материнская плата** — соединяет все компоненты между собой
- **GPU** — специализированный процессор для работы с графикой
- **Блок питания** — преобразует напряжение из розетки и питает все компоненты

**Что происходит при двойном клике на программу:**

1. ОС находит файл программы на диске
2. Создаёт новый процесс
3. Копирует код программы с диска в RAM
4. CPU начинает исполнять инструкции из RAM

**Одно ядро vs четыре ядра:**

Одно ядро выполняет один поток инструкций за раз. Четыре ядра — четыре потока одновременно. Для веб-сервера это критично: с четырьмя ядрами он может обрабатывать четыре запроса параллельно вместо одного.

## Оперативная память и жёсткий диск

**Почему нельзя обойтись только диском:**

Диск (HDD/SSD) в тысячи раз медленнее RAM. Если CPU читал бы данные напрямую с диска — компьютер работал бы невыносимо медленно. RAM — это буфер между быстрым CPU и медленным диском.

**Почему нельзя обойтись только RAM:**

RAM энергозависима — при выключении все данные теряются. Хранить файлы, ОС и программы негде. Плюс RAM дорогая и её объём ограничен.

**Когда программа загружается с диска в RAM:**

В момент запуска. ОС читает исполняемый файл с диска и копирует его в RAM. CPU не может исполнять инструкции прямо с диска — только из RAM.

**При выключении компьютера:**

Всё содержимое RAM стирается — электрические заряды исчезают без питания. Поэтому несохранённые документы пропадают.

## Процессы и потоки

**Программа vs процесс:**

Программа — файл на диске. Процесс — запущенная программа в памяти. Один файл можно запустить несколько раз — получатся отдельные процессы, каждый со своей памятью.

**Два окна браузера:**

Зависит от браузера. Chrome создаёт отдельный процесс на каждую вкладку. Два окна — как минимум два процесса, а то и больше.

**Поток vs процесс:**

Процесс — изолированный контейнер с памятью и ресурсами. Поток — единица исполнения внутри процесса. Потоки одного процесса делят общую память, поэтому создаются быстрее и общаются проще — но это же делает их опасными.

**Два потока и одна переменная — race condition:**

Оба потока могут прочитать значение `counter = 5` одновременно, оба прибавят 1 и оба запишут `6`. А должно было получиться `7`. Результат непредсказуем и зависит от порядка выполнения — это и есть гонка данных.
